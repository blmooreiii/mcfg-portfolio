i.
The Problem

Survivors need help in seconds.
Existing tools take minutes.

In a moment of crisis, the difference between a shelter found and a chance lost can be a single screen. Today's apps add too many.

Domestic violence survivors in crisis need immediate access to shelter and support. Existing tools are too complex to navigate under stress, require account creation, or leave identifiable traces on shared and monitored devices.

The brief was clear: build something that works in seconds, leaves no trace, and feels nothing like a traditional help app.

Every conversation with advocates pointed back to the same constraint — privacy is not a feature you add late. It's the only frame the product can be designed inside.

Victims need a way to find safe shelters and support resources immediately, without exposing themselves to risk or navigating complex existing systems.

— Project Brief
ii.
Principles & Posture

Four commitments,
locked at the outset.

Each principle is a veto. If a feature breaks one, the feature does not ship — full stop.

— i. —

Trust Is the Product

Every decision evaluated against whether it builds or breaks user trust. Nothing else compounds without it.

— ii. —

Privacy Is Table Stakes

No account, no history, no stored location, no trace. The default is silence.

— iii. —

Discretion Over Visibility

App icon designed to pass as a casual game on a home screen. Disguise is a safety feature.

— iv. —

Navigation Tool Only

No chat, no push notifications, no community features. Find what's needed, then leave.
